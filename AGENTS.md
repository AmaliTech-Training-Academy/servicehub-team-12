# Repository Instructions

## Commit Messages

- Use Conventional Commits for all commit messages.
- Include the GitHub issue number in parentheses at the end of the subject when an issue is being addressed, for example: `(#36)`.
- For frontend-related changes, use the `frontend` scope, for example: `feat(frontend): update login form styling (#36)`.
- Keep the commit subject in the format `<type>(<scope>): <summary> (#issue-number)` when a scope and issue number apply.
- Common commit types include `feat`, `fix`, `refactor`, `docs`, `test`, and `chore`.
